# AXIOM-Mobile Results

This directory stores experiment artifacts that are generated locally and are not intended to be committed in full.

## Baseline Runs

The Phase 2 scaffold writes lightweight baseline runs to:

- `results/baselines/<model_id>_seed<seed>/`

Current executable baseline:

- `question_lookup_v0`

Expected files per run:

- `run_result.json`
- `model_state.json`
- `predictions_pool.jsonl`
- `predictions_val.jsonl`
- `predictions_test.jsonl`

`run_result.json` is the canonical summary artifact for later analysis.

## Selection Sweeps

Phase 3 sweep results are written to:

- `results/selection_sweeps/sweep_v0/`

See `docs/SELECTION_STRATEGIES.md` and `docs/LEARNING_CURVES.md` for details.

## Device Profiles

Phase 5 on-device profiling sessions are stored under:

- `results/device_profiles/{device}_{model_id}_{timestamp}/`

Each session directory contains the app-exported CSV + `_meta.json`, plus optional `trace_metrics.json` (manually extracted Instruments data) and raw `.trace` files.

The summary pipeline (`ml/scripts/summarize_device_profiles.py`) ingests sessions and writes analysis artifacts to `results/device_profiles/analysis/`.

Synthetic test fixtures live in `results/device_profiles/_fixtures/` and are not production data.

## Statistical Analysis

Phase 6 analysis outputs are written to:

- `results/analysis/phase6_v0/`

Generated by `ml/scripts/run_statistical_analysis.py`, which ingests baselines, selection sweeps, CoreML exports, and device profiles.

| File | Description |
|------|-------------|
| `summary.json` | Complete machine-readable analysis (~19KB) |
| `summary.csv` | Flat model comparison table |
| `analysis.md` | Human-readable report with tables, fits, and caveats |
| `learning_curves.svg` | Deterministic learning-curve plot (no matplotlib) |

**Current status (phase6_v0):** `partial` — 3 strategies analyzed across 6 budgets, 2 simulator sessions, 0 physical-device sessions. All results carry explicit status labels. See `docs/STATISTICAL_ANALYSIS.md` for methodology.

## Paper Assets

Paper-ready assets are generated into `paper/assets/generated/` by:

```bash
python3 ml/scripts/build_paper_assets.py
```

This reads from the analysis and device-profile outputs above and writes deterministic SVG, CSV, and Markdown files. See `paper/README.md` for details.

### Simulator sessions

Two simulator sessions validate the profiling pipeline (not publishable):

1. `iphone17pro-sim_tiny_multimodal_v0_20260412T212027Z/` — 20 iterations, Debug build, blank input. p50=199.5ms, p95=304.2ms (PASS).
2. `iphone17pro-sim_tiny_multimodal_v0_20260413T034245Z/` — 50 iterations, Release build, synthetic test pattern (`image_loaded=true`). p50=98.0ms, p95=112.8ms (PASS). Includes `trace_metrics.json` sidecar and xctrace validation.

### Staging new sessions

Use the staging script to copy app exports into the correct folder structure:

```bash
python3 ml/scripts/stage_device_profile_session.py \
    --from-simulator --device-name "iphone17pro-sim"
```

See `docs/DEVICE_PROFILES.md` for the full session contract, schema, and threshold evaluation documentation.
